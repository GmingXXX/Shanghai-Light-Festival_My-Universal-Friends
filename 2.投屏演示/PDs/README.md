# 海洋生物动态壁纸

一个纯前端的多视频动态壁纸网页，可以同时展示多个透明背景的海洋生物视频，让它们在屏幕上自由漂浮游动。

## 功能特点

- ✅ **高性能渲染**：使用HTML5 Canvas技术，可同时流畅显示80+个视频
- ✅ **自然运动**：海洋生物以极其缓慢的速度游动，模拟真实海洋环境
- ✅ **边界反弹**：碰撞到屏幕边界时自然反弹，继续游动
- ✅ **相互穿透**：视频之间可以相互穿过，不会发生碰撞
- ✅ **拖拽上传**：支持直接拖拽webm文件到网页中添加新的海洋生物
- ✅ **宇宙背景**：使用background文件夹中的UNIVERSE.mp4作为动态背景
- ✅ **纯前端**：单一HTML文件，无需任何环境配置即可使用

## 使用方法

### 基本操作

1. **打开网页**：用现代浏览器（Chrome、Firefox、Edge等）打开 `index.html` 文件
2. **添加海洋生物**：
   - 点击右上角的 "+" 按钮选择webm视频文件
   - 或直接将webm文件拖拽到网页中
3. **观看效果**：海洋生物会自动开始在屏幕上缓慢游动

### 键盘快捷键

- **H键**：显示/隐藏帮助面板
- **C键**：清除所有海洋生物
- **R键**：重置画布
- **ESC键**：隐藏帮助面板

## 文件要求

### 视频格式
- **格式**：必须是 `.webm` 格式
- **背景**：必须是透明背景（带Alpha通道）
- **大小**：单个文件不超过50MB
- **数量**：最多同时显示100个视频（推荐80个以内以保证流畅性）

### 背景视频
- 将背景视频命名为 `UNIVERSE.mp4` 并放置在 `background/` 文件夹中
- 或修改HTML文件中的视频路径

## 性能优化

### 系统要求
- 现代浏览器（支持HTML5 Canvas和Video API）
- 建议使用独立显卡以获得更好的性能
- 8GB以上内存推荐

### 性能提示
- 视频数量超过20个时，系统会自动监控FPS
- 当FPS低于30时，会显示性能警告
- 建议视频分辨率不超过500x500像素
- 关闭其他占用GPU的程序以获得最佳性能

## 故障排除

### 常见问题

1. **视频不显示**
   - 确保视频格式为webm且带有透明背景
   - 检查浏览器是否支持webm格式

2. **性能卡顿**
   - 减少海洋生物数量
   - 降低视频分辨率
   - 关闭其他浏览器标签页

3. **背景视频不播放**
   - 确保UNIVERSE.mp4文件在background文件夹中
   - 检查视频文件是否损坏

4. **拖拽上传无响应**
   - 确保拖拽的是webm格式文件
   - 检查浏览器控制台是否有错误信息
